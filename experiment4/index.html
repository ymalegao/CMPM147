Experiment 3 - Alternate Worlds

Description

We are trying to generate and implement autotiling so we can automatically generate maps and dungeons. Users can edit the map from the ascii grid on the left

Technical

We first have a grid that is filled with one code, and that grid is populated and over written using noise and random to generate biomes. Once that is done, I implemented auto-tiling logic to make sure that the tile to tile connection would be smoother. To do this we used tile bitmasking and a look up table, that was manually filled out in order to provide the right tile in the right context. It was like Wave Collapse Function but a little easier.

Reflection

For me the hardest part was the look up table and understanding how everything worked considering the autotiling. I was stuck on how to generate biomes and then apply the auto tiler to that until I figured out that you had to layer the codes so that they would render on top of one another. Once I had figured that out, there were a few edge cases that had to be fixed, but other than that, I think working with noise and random is not the easiest task in the world, espcially since they are so random! My highs were figuring out the autotiling logic, making the clouds not spawn in the same spot over and over, which was happening because of the random seed that was set.

Artistic Statemnt
